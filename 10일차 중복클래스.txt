익명 구현 객체에서 바깥의 데이터(for(int i))를 접근할때 final 타입의 변수를 선언하여 사용해야된다.
데모 코드 : 기능이 하나밖에 없는 정적 코드
중첩 멤버 클래스 : 인스턴스 멤버 클래스는 바깥 클래스의 객체가 있어야 사용가능
정적 멤버 클래스는 바깥 클래스의 객체가 없어도 사용가능
인스턴스 멤버 클래스 내부에는 바깥 클래스의 모든 필드와 메소드 사용 가능
로컬 클래스 : 로컬 클래스는 메소드 내부에 선언된 클래스
- 로컬 클래스는 바깥 클래스의 필드와 메소드를 사용 가능
- static 키워드를 시용해 정적 클래스를 만들수 없음
- final 특성을 가진 매개변수나 로컬변수만을 로컬 클래스 내부에서 사용 가능
익명 객체 : 클래스를 상속하거나 인터페이스를 구현해야만 생성 가능
익명 객체는 필드, 매개변수, 로컬 변수의 초기값으로 구조 사용
익명 객체는 주로 재정의된 메소드를 멤버로 가짐
